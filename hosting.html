KidsApp Hosting Guide

This guide explains how to host your backend and frontend from GitHub to Render.

1. Project Structure

kidsapp/
  backend/
    server.js
    package.json
    .env
    routes/
  frontend/
    package.json
    vite.config.js
    src/

Your GitHub repo root should be the kidsapp folder.

2. Push Project to GitHub

  1. Open terminal inside e:\kidsapp6\kidsapp
  2. Run these commands:
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/yourname/kidsapp.git
git push -u origin main

3. Backend Hosting on Render

  1. Go to Render Dashboard
  2. Click New +
  3. Select Web Service
  4. Connect GitHub and choose the same repo

Backend Settings

  • Name: kidsapp-backend
  • Root Directory: backend
  • Environment: Node
  • Build Command: npm install
  • Start Command: node server.js

Backend Environment Variables

GEMINI_API_KEY=your_key_here
APP_PASSWORD=your_app_password
JWT_SECRET=your_secret_here

After deployment, Render gives a backend URL like:

https://mykidsapp.onrender.com

4. Frontend Hosting on Render

  1. Go to Render Dashboard
  2. Click New +
  3. Select Static Site
  4. Select the same GitHub repo

Frontend Settings

  • Name: kidsapp-frontend
  • Root Directory: frontend
  • Build Command: npm install && npm run build
  • Publish Directory: dist

Frontend Environment Variable

VITE_API_BASE_URL=https://mykidsapp.onrender.com

This connects your frontend to the backend hosted on Render.

5. Important Notes

  • Both frontend and backend use the same GitHub repo.
  • Backend uses Root Directory = backend.
  • Frontend uses Root Directory = frontend.
  • localStorage data is stored in the browser, not on Render.
  • Do not push backend/.env to GitHub.

6. Example .gitignore

node_modules/
backend/.env
backend/node_modules/
frontend/node_modules/
frontend/dist/

7. Final Flow

  1. Push code to GitHub
  2. Deploy backend on Render as Web Service
  3. Copy backend URL
  4. Deploy frontend on Render as Static Site
  5. Set VITE_API_BASE_URL to backend URL
  6. Open frontend Render URL and test the app
